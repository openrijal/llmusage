# Implementation Details - Table UI Enhancements

## Overview

Enhance the daily/weekly/monthly table rendering in `src/display.rs` to support:
1. Dynamic column widths based on content
2. Shorter model names with provider-aware abbreviation
3. Provider grouping with colored provider labels and aggregate values
4. Per-model token and cost breakdown under each provider
5. Dotted line separators between models within a provider
6. Proper date column width for full YYYY-MM-DD
7. `--all` flag to show/hide zero-value models
8. Separators between time periods
9. Consistent alignment across all modes

## File Changes

### src/models.rs
- Added `ModelEntry` struct with `provider`, `model`, `input_tokens`, `output_tokens`, `cost`
- Added `model_entries: Vec<ModelEntry>` field to `DailyRow`

### src/db/queries.rs
- Modified `query_grouped()` to SELECT `provider` alongside `model`
- GROUP BY now includes `provider` for per-provider-per-model breakdown
- Each model entry includes its individual cost from `COALESCE(SUM(cost_usd), 0)`
- Results populate both legacy `models: Vec<String>` and new `model_entries: Vec<ModelEntry>`

### src/main.rs
- Added `--all` flag (bool) to `Daily`, `Weekly`, `Monthly` command variants
- Updated `cmd_daily()`, `cmd_weekly()`, `cmd_monthly()` signatures to accept `show_all`
- Passes `show_all` through to `display::print_daily()`

### src/display.rs

#### New structures
- `DisplayRow` — holds label, input, output, cost, and `RowKind`
- `RowKind` enum — `Provider` (aggregate), `Model` (per-model detail), `DottedSep` (separator)
- `build_period_rows()` — builds display rows for one period, grouped by provider

#### Provider grouping with aggregates
Each period now renders as:
```
│ 2026-04-04 │ claude_code     │        3 │      13 │    $0.271 │  ← provider aggregate (magenta)
│            │   opus-4-6      │        3 │      13 │    $0.271 │  ← per-model (dimmed)
│            │ codex           │ 2,025,711│  36,927 │    $0.00  │  ← another provider
│            │   codex-openai  │ 2,025,711│  36,927 │    $0.00  │  ← per-model
```

#### Dotted separators
Between models within a provider (when >1 model), a dotted line is rendered using `·` characters:
```
│            │ ···················│ ··········│ ········│ ···········│
```

#### Model-level dimming
Per-model values are rendered with `.dimmed()` to visually distinguish them from provider aggregates.

#### Dynamic column widths
Column widths are computed by scanning ALL display rows (provider + model lines) across ALL periods, plus header labels and totals, to find the maximum width needed.

#### shorten_model() enhancements
- Strips `antigravity-` prefix (for opencode models like `antigravity-gemini-3-flash`)
- Strips `claude-` prefix
- Strips date suffixes (`-YYYYMMDD`)
- Strips `4-` prefix variant

#### Single-line dates
- Daily: `2026-04-15` (no multi-line split)
- Weekly: `2026 W15` (space instead of newline)
- Monthly: `2026-04` (unchanged)

## Design Decisions

### Provider aggregate = sum of model entries
Provider row values are computed by summing all `ModelEntry` values for that provider within the period. This matches the period total when there's only one provider.

### --all flag behavior
- Default (no flag): filters out `ModelEntry` where both `input_tokens == 0` AND `output_tokens == 0`
- `--all`: shows all models regardless of token values
- Currently no models in the dataset have zero tokens, so the filter has no visible effect, but it's correctly implemented for when such data exists

### Dotted separator only between models
Dotted lines appear between models within the same provider. They do NOT appear between providers (solid separators handle period boundaries).

### Backward compatibility
- `models: Vec<String>` field retained in `DailyRow` for JSON output compatibility
- `model_entries` added as new field, visible in `--json` output with full per-model detail
