# Implementation Details - PR #68: feat/gemini-cli-jsonl-support

## Overview

PR #68 replaced the stub Gemini CLI collector with a working JSONL parser and expanded `.gitignore`. The key insight was that Gemini CLI stores sessions in JSONL format (not protobuf), making parsing possible.

## Gemini CLI Session Format Discovery

**New path**: `~/.gemini/tmp/<project-hash>/chats/*.jsonl`

**Log format** (based on gemini-cli `chatRecordingTypes.ts`):
```
{"sessionId":"sess-1", ...}           // Session metadata
{"id":"m1","type":"user",...}          // User message (skip)
{"id":"m2","type":"gemini","model":"gemini-2.5-pro","tokens":{"input":120,"output":40,"cached":10,"total":175,"thoughts":5,"tool":0}}
{"$rewindTo":"m1"}                     // Rewind command
```

Key fields in `TokensSummary`:
- `input` - prompt tokens
- `output` - completion tokens
- `cached` - cached token count
- `total` - sum (used for validation)
- `thoughts` - thinking/reasoning tokens (optional)
- `tool` - tool use tokens (optional)

## Parsing Challenges and Solutions

### Challenge 1: Duplicate Message Writes
Gemini CLI appends each message twice:
1. First write: without tokens (tokens field absent)
2. Second write: with tokens populated

**Solution**: Only process lines with actual `tokens` data. The `messages.insert()` in `HashMap` overwrites the first empty entry.

### Challenge 2: Session Rewinding
`$rewindTo` records truncate the session history, removing messages after the rewind point.

**Solution**: When `$rewindTo` is encountered, find and remove messages after that ID using the `order` Vec index. If ID not found, clear everything.

### Challenge 3: Subagent Sessions
Subagent sessions live in subdirectories under the parent chat directory.

**Solution**: `collect_chats_dir()` recursively descends into subdirectories, treating any JSONL file found as a valid session.

### Challenge 4: Zero-Token Messages
Some messages have all-zero token counts.

**Solution**: Skip records where `input == 0 && output == 0 && cached == 0 && total == 0`.

## Legacy Protobuf Detection

Gemini CLI previously used `~/.gemini/antigravity/conversations/*.pb` (protobuf format). These files are opaque without the schema.

**Detection logic**: Check for `.pb` files in antigravity directories. If found but no JSONL sessions exist, print an upgrade warning directing users to start a new session.

## Collector Auto-Detection Update

Changed detection from:
```rust
gemini_dir.join(".gemini").join("antigravity").join("conversations").exists()
```

To:
```rust
jsonl_dir.join(".gemini").join("tmp").exists() || legacy_dir.join("antigravity").exists()
```

This ensures the collector activates for either new JSONL sessions or legacy protobuf files (to show the warning).
